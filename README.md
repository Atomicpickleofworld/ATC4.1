# 🔐 Atomic TriFlow Cipher v4.1 — Stable & Secure Edition

Современный десктопный шифратор с модульной архитектурой, мультиблочным шифрованием и разделённым хранением ключей. Написан на Python 3 + PyQt6.

> ✨ **Версия 4.1** — стабильный релиз с исправленной вёрсткой, потоковой обработкой и усиленной криптографической обвязкой (AAD, HMAC, anti-timing).

## 🛡️ Криптографическая схема

| Компонент              | Реализация                          | Назначение                                  |
|------------------------|-------------------------------------|---------------------------------------------|
| **KDF**                | `Argon2id` (fallback: PBKDF2-SHA256)| Вывод ключа из пароля, устойчивость к GPU/ASIC |
| **AEAD Шифр**          | `ChaCha20-Poly1305` (fallback: AES-GCM) | Аутентифицированное шифрование              |
| **AAD-привязка**       | `VERSION(4B) + BLOCK_IDX(4B) + TIMESTAMP(8B)` | Защита от перестановки, downgrade и replay-атак |
| **Целостность заголовка** | `HMAC-BLAKE2b(32B)`              | Обнаружение подмены формата или обрезки     |
| **Защита от timing**   | Dummy KDF при проверке длины пароля | Предотвращение атак по времени исполнения   |

### 🔑 Мультиблочная архитектура
1. Текст разбивается на блоки случайной длины (`64–256` байт)
2. Каждый блок шифруется **своим уникальным ключом**
3. Ключи хранятся в отдельной кодбазе (`.atcdb`)
4. Шифртекст (`.atc4`) и кодбаза передаются **разными каналами**
5. Без кодбазы расшифровка математически невозможна

## 📖 Использование

### 🔒 Шифрование
1. Введи текст в поле или нажми `📂 Обзор` и выбери `.txt/.md/.json` файл
2. (Опционально) Включи `Защитить кодбазу мастер-паролем` и введи пароль (мин. 16 символов)
3. Нажми `🔒 Зашифровать`
4. Сохрани оба файла через кнопки `💾 Сохранить .atc4` и `💾 Сохранить .atcdb`
5. **Передай файлы разными каналами связи**

### 🔓 Дешифрование
1. Выбери файл `.atc4` и соответствующий `.atcdb`
2. Если кодбаза защищена, введи мастер-пароль
3. Нажми `🔓 Расшифровать`
4. Результат появится в поле, можно скопировать или сохранить в `.txt`

## 📄 Формат файлов

### `.atc4` (Шифртекст)
```
MAGIC(4) + VERSION(1) + GLOBAL_SALT(16) + TIMESTAMP(8) + BLOCK_COUNT(4)
+ для каждого блока: LEN(4) + [SALT(16) + NONCE(12) + TAG(16) + CT_LEN(4) + CIPHERTEXT]
+ HEADER_HMAC(32)
```

### `.atcdb` (Кодбаза)
```json
{
  "version": "4.1",
  "format": 4,
  "global_salt": "hex...",
  "timestamp": "hex...",
  "block_count": 12,
  "blocks": [
    {"idx": 0, "key": "str...", "key_len": 128, "blk_len": 128, "timestamp": "ISO8601"},
    ...
  ],
  "protected": false
}
```
Если `protected: true`, JSON зашифрован через `Argon2id → ChaCha20-Poly1305` с AAD `b'ATCDB_v41'`.

## ⚠️ Правила безопасности

1. 🔐 **Никогда не передавай `.atc4` и `.atcdb` одним каналом** (разные мессенджеры, почта + флешка и т.д.)
2. 📉 **Потеря любого из файлов = безвозвратная потеря данных**. Резервные копии храни отдельно.
3. 🔑 **Минимальная длина пароля: 16 символов**. Система принудительно проверяет это перед KDF.
4. 🚫 Не используй в production-критичных системах без независимого криптографического аудита.
5. ️ Кодбаза не содержит никаких следов исходного текста. Утечка `.atc4` без `.atcdb` безопасна.
